# Industry Cognition Skill

一个面向策略运营和行业研究的 Codex skill。它帮助 AI 先定义行业，再用上市龙头、招股书、年报、极简财务模型、创业者和投资人信号来形成行业认知。

它不是新闻摘要器，也不是运营套路库。它更适合回答这类问题：

- 外卖到底是一门什么生意？
- 餐饮供应链应该被理解成批发分销、餐饮工业化，还是连锁餐饮基础设施？
- 一个新 AI 产品或融资事件，是否真的改变了行业的核心变量？
- 招股书和年报里哪些数字能支撑市场判断和业务模型？

## Core Idea

默认研究顺序：

```text
定义行业 -> 找龙头参照 -> 读招股书/财报 -> 建极简模型 -> 用创业/投资信号更新认知
```

核心品味：

- 内核稳定：商业本质、成本结构、利润池、竞争格局不会因为热点轻易改变。
- 拥抱趋势：新技术、产品形态、履约方式和分发路径可能改变行业边界。
- 龙头叙事优先：行业定义需要从代表性公司在招股书、年报和投资者材料里的叙事中反推。
- 数据进入公式：关键数字不应只是事实表，而要能代入业务模型。

## Install

把这个仓库放到本地 Codex skills 目录：

```bash
git clone git@github.com:shawting7/industry-cognition-skill.git ~/.codex/skills/industry-cognition
```

如果已经存在同名 skill，可以先备份旧目录再替换。

## Usage

在 Codex 里直接提问即可，例如：

```text
用 $industry-cognition 研究餐饮供应链
```

或：

```text
用 $industry-cognition 跑一次外卖研究协议
```

默认输出会尽量保持紧凑，不写成百科式行业报告。

## What It Produces

通常会输出：

- 主问题：这个行业到底是什么生意？
- 候选定义：每个定义对应哪些关键指标？
- 龙头叙事：上市公司如何描述自己和行业？
- 招股书/财报四件事：宏观市场、用户需求、竞争环境、极简模型。
- 极简模型代入：选一家锚点公司，把真实数字放进公式。
- 验证问题：用更多龙头和相邻公司验证判断。
- 趋势更新：创业、融资、投资人、产品 demo 只作为模型变量更新。

```text

│   └── openai.yaml
└── references/
    ├── filings-reading.md
    └── food-delivery.md
```

`SKILL.md` 是 skill 的主入口。`references/` 只在需要时加载，用来保存招股书/年报读法和外卖标准样例。

## Notes

这个 skill 的目标不是让 AI 显得很会套框架，而是让它少说空话，多围绕行业定义、龙头参照、关键数字和商业变量形成判断。
